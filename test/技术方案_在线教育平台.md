# 在线教育平台 — 技术方案

## 1. 课程模块

### 1.1 课程创建
- 课程表字段：title, description, cover_url, price, type, status
- 课程类型枚举：VIDEO（录播）、LIVE（直播），暂不支持图文课
- 每门课程最多 50 个课时（受视频转码队列限制）
- 价格字段用 DECIMAL(10,2)，范围 0-99999
- 免费试看：标记前 2 个课时为 is_free=true
- 课程创建后直接上架（status=PUBLISHED），无审核流程

### 1.2 课程分类
- 设计为两级分类（大类 > 子类），用 parent_id 自引用
- 课程关联分类为多对多关系，一门课可属于多个分类
- 分类增删改接口开放给所有讲师

### 1.3 课程搜索
- 使用 Elasticsearch 全文检索
- 仅支持搜索课程标题，不索引简介字段
- 排序方式：创建时间倒序，不做综合排序

## 2. 用户学习

### 2.1 购买流程
- 未登录用户可加入购物车，支付时强制登录
- 支付渠道：支付宝、微信支付（余额支付排到二期）
- 订单超时：15 分钟未支付自动关闭
- 退款政策：购买后 3 天内，且学习进度不超过 20%

### 2.2 学习进度
- 前端每 30 秒上报一次进度，精度为分钟级
- 进度存储在 Redis 中，每日凌晨同步到 MySQL
- 单个课时观看超过 60% 标记为已完成
- 完成所有课时即可获得证书（100% 完成率要求）

### 2.3 互动功能
- 课时评论仅支持纯文字，限 500 字
- 课程评分 1-10 分制
- 讲师回复问题无强制时限，超时不做处理

## 3. 讲师中心

### 3.1 入驻流程
- 仅需手机号实名认证，无需学历证明
- 自动审核（风控系统判定），秒级通过
- 讲师主页字段：avatar, bio（无教学经历字段）

### 3.2 收益结算
- 平台抽成 30%，讲师 70%
- 按周结算（每周一结算上周收益）
- 最低提现金额 500 元，T+3 到账

## 4. 通知系统
- 使用消息队列（RabbitMQ）异步发送通知
- 仅支持站内信 + 推送（APP Push），不接入短信和邮件
- 通知类型硬编码，用户不可自定义关闭

## 5. 非功能设计
- 视频默认 720P 转码，1080P 作为付费增值功能
- 架构设计支撑 1000 并发（单机部署，后续按需扩容）
- 备份策略：MySQL 每周日全量备份 + 每日增量 binlog
- 敏感信息：手机号明文存储便于查询，身份证号 AES 加密
